# Git进阶笔记

## 综述：

初接触的时候真的是啥都不会，现在研究来研究去总算搞懂了点。在我的第一次git学习笔记里我提到了初始化本地仓库、暂存文档等等操作，当时只是在借助ai一步步完成过程上传文件，现在才开始逐渐了解。个人觉得git里很多指令听起来真的太高大上，让初学者觉得很难都不敢去接触，比如暂存文件、版本链啥的，但只有坚持去接触了才发现其实就是名字高贵，根本思路还是那些。



## 一、学习到的基础指令

1.初始化:听起来很牛x，但其实就是在目录里搞一个新的git仓库用于存储后面准备上传的代码罢了。写上“git init”就行。但我个人觉得还不如直接在桌面右键新建后续在读取这个文件夹的地址方便。

2.添加与删除文件：用”git add“+文件名或者文件夹就能选是添加整个文件还是单个，不想要了就输入”git reset HEAD“,如果加的是文件名就是单个，加”.“就是全部暂存的都删了。

3.查看历史版本：既然与Github有关，那肯定是可以调用库来看历史版本的。

如果只是想看单个版本： git log（详细）

​                                            git log--online（简洁）

如果想看总的并且还有便捷的需求：git log --graph --oneline

4.关于分支：这是在用vs code上传的时候乱搞摸索出来的（个人觉得vs比git bash方便太多了，直接翻就可以找，git bash里还要去输入）

如果想要查看：git branch(本地的)

​                           git branch -r远程的）

​                           git branch -a(全部的)

如果想要创建：git branch+名字

如果想要切换：git chackout/switch+名字

如果想要删除：git branch -d(如果是大写D就是强制删除未合并的，慎用！之前没搞懂用了这个全给它搞没了)

5.撤销与回滚：这两个简直就是git命名规则高大上的典型，我开始看了以为是什么根本学不会的操作，其实就是相当于悔棋和掀桌不下了回到之前的棋局。

撤销：git checkout --文件名或git restore 文件名

回滚：git reset --hard 版本哈希值



and so on

## 二、利用VS code提交代码

### 总述：

刚开始在用Git bash的时候我都是一头雾水，不懂怎么用。用了ai后熟了一些步骤，才慢慢开始打自己的命令，但每次上交代码到库里还是很麻烦。然后在翻任务要求的时候看到有建议用VS code，正好之前就下了，放了好久也不懂怎么用，就想着是不是和这有关，然后就去b站搜了教程，借助视频总算弄清楚了。而且vs code也更好看，git bash太老了，用久了看的都力歇/、。（参考文献：https://www.bilibili.com/video/BV1Hkr7YYEh8/?spm_id_from=333.1391.0.0&vd_source=4f6e872bba42009b9c587cefecf237ba）



### 步骤

1.不必多说，先创立好仓库，还有将git绑定相应的名字邮箱

2.写完代码后点commit，后面会出现publish branch的字样，点，然后就会跳转出授权的页面

3.回到vs code会出现选公开还是私人（上传需要挂梯子）

ps：我个人在上传的时候因为校园网卡的要死，ai就建议我把http改成ssh，然后相应的步骤和git上传没有太大区别
